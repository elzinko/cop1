# Story EA14-S3: Supervisor prompt commit_anchor guidance

Status: done

## Story

As a **cop1 supervisor orchestrating a dev-story session**,
I want the supervisor system prompt to include explicit guidance about invoking the `commit_anchor` tool after successful implementation,
so that the LLM knows to anchor completed work in git history with a `Co-Authored-By` trailer.

## Acceptance Criteria

1. **AC1 — Prompt includes commit_anchor guidance for dev-story**
   - When `workflowCommand` contains `dev-story`, the assembled supervisor prompt includes a "Commit Anchor" section instructing the LLM to call `commit_anchor` after successful implementation.
   - The guidance mentions the `message` parameter, the `worktreePath` optional parameter, and the `nothing_to_commit` fallback.

2. **AC2 — Prompt does NOT include commit_anchor guidance for other commands**
   - When `workflowCommand` is `/bmad-bmm-code-review`, `/bmad-bmm-create-story`, or any command not containing `dev-story`, the prompt does not mention `commit_anchor`.

3. **AC3 — Tool is already registered in MCP server**
   - Verified: `commit_anchor` is registered in `SupervisorMcpServer.ts` (line 62-69, delivered in EA12-S1). No wiring change needed.

4. **AC4 — Tests cover both branches**

     - dev-story without slash prefix -> prompt contains commit_anchor guidance
     - code-review command -> prompt does NOT contain commit_anchor guidance
     - create-story command -> prompt does NOT contain commit_anchor guidance

5. **AC5 — Build green, existing tests pass**

## Dev Notes

### Investigation findings

The architecture has three layers relevant to commit_anchor:

1. **MCP tool catalog** (`toolCatalog.ts` + `SupervisorMcpServer.ts`): `commit_anchor` IS registered as an MCP tool in the supervisor's tool server. This was delivered in EA12-S1 and works correctly.

2. **Supervisor prompt** (`SupervisorPromptBuilder.ts`): The system prompt for the supervisor LLM did NOT mention `commit_anchor`. Without prompt guidance, the LLM has no reason to call the tool even though it exists in the catalog.

3. **Session adapter** (`AgentSdkSessionAdapter.ts`): The BMAD session runs Claude Code with standard tools (Bash, Read, Write, etc.) but does not expose MCP tools directly. The `commit_anchor` tool is available through the supervisor's MCP server, which is the correct invocation path.

### Implementation choice

Option A (prompt guidance only) was chosen because:
- The tool is already registered in the MCP server (no Option B needed).
- The guidance is conditional on `workflowCommand.includes('dev-story')` so it only appears for dev-story sessions, not for code-review or create-story.
- The guidance is placed between the Scrum Cycle Guidance and Story Content sections, giving the LLM context about when and how to call the tool.

### Why not add to supervisor-playbook.md

The playbook is intent-only (per EA12-S3 A5 pivot) and should not enumerate specific tool invocations. The tool guidance belongs in the system prompt that the LLM receives, where it has programmatic control over conditional inclusion.
