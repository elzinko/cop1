# Story EA12.5: Runtime hardening — budget simplification + reentrance guard (A1+A4)

Status: todo

## Story

As a **cop1 orchestrator runtime**,
I want the budget model simplified to a single `max_tokens_per_night` cap and the reentrance guard to return structured errors instead of throwing,
so that the runtime is simpler to configure and errors are recoverable by the supervisor.

## Acceptance Criteria

1. **AC1 — Single budget cap**
   - Remove the 3-cap structure (`max_turns_per_workflow`, `max_tokens_per_session`, `max_duration_per_workflow_seconds`).
   - Replace with `max_tokens_per_night` (default 2,000,000). Rely on native SDK compaction for session-level management.

2. **AC2 — remaining_budget fail-fast**
   - `remaining_budget` tool in `toolCatalog.ts` must fail-fast if `budgetProvider` is absent — no `Number.POSITIVE_INFINITY` fallback.
   - Returns structured error `{ error: 'no_budget_provider' }`.

3. **AC3 — Reentrance guard configurable**
   - `cap=3` default, configurable via `budgets.max_reentrance_depth` in playbook frontmatter.
   - Playbook schema updated to accept this optional field.

4. **AC4 — Structured reentrance error**
   - Replace `throw new Error(...)` at `toolCatalog.ts:~81` with structured return: `{ error: 'reentrance_cap', escalation_required: true, depth, max }`.
   - Supervisor can read the error and escalate gracefully.

5. **AC5 — Track 3 event**
   - New event type `reentrance.cap_hit` added to the Track 3 metrics catalogue.
   - Emitted when reentrance cap is reached.

6. **AC6 — Tests**
   - Unit tests for budget fail-fast, reentrance structured error, configurable depth.
