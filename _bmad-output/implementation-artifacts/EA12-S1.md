# Story EA12.1: commit_anchor real implementation

Status: done

## Story

As a **cop1 orchestrator runtime**,
I want `commit_anchor` to produce real git commits (conventional format, Co-Authored-By trailer, fail-safe on dirty worktree),
so that SessionLogAggregator (D4) can reference real commit SHAs and the supervisor's work is anchored in git history.

## Acceptance Criteria

1. **AC1 — Real commit produced**
   - `commit_anchor` tool in `toolCatalog.ts` replaces the V1-light stub (lines 122-128) with a real `git commit` invocation.
   - Commit message follows conventional commit format: `<type>(<scope>): <description>`.
   - Commit includes `Co-Authored-By: <supervisor-identity>` trailer.

2. **AC2 — Fail-safe on dirty worktree**
   - If the worktree has no staged changes, `commit_anchor` returns `{ committed: false, reason: 'nothing_to_commit' }` — no error thrown.
   - If `git commit` fails for any reason, returns structured error `{ committed: false, reason: 'commit_failed', detail: <stderr> }`.

3. **AC3 — No auto-push**
   - `commit_anchor` MUST NOT push to remote. Push is a separate, user-initiated action.

4. **AC4 — SHA returned**
   - On success, returns `{ committed: true, sha: '<full-sha>', short_sha: '<7-char>' }`.
   - SHA is the real commit hash from `git rev-parse HEAD` after commit.

5. **AC5 — Unit + integration tests**
   - Unit test with mocked git commands verifying all return shapes.
   - Integration test in a temp git repo verifying a real commit is produced.
