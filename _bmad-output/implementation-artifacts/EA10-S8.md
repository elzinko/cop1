# Story EA10.8: Supervisor multi-step resolution loop

Status: done

## Story

As a **cop1 Supervisor receiving a blocked state from `BMADSessionPort`**,
I want to extend `SupervisorService` from its current 3-tier cascade (deterministic → LLM → terminal escalation) to a multi-step loop (deterministic → LLM → consult agents via EA10-S7 → synthesize consensus → escalate if all fail),
so that the orchestrator has a realistic chance of resolving intermediate blocks without human escalation, while keeping escalation easy and traceable when the loop genuinely cannot proceed.

## Acceptance Criteria

1. **AC1 — State machine**
   - `SupervisorService` exposes a state machine with states: `idle`, `deterministic`, `llm`, `blocked`, `consulting`, `synthesizing`, `escalated`
   - Transitions documented as a diagram in Dev Notes and as a typed enum in code

2. **AC2 — Resolution phases in order**
   1. **Deterministic** — rule-based match against the question (continues existing EA9-S3 behavior)
   2. **LLM** — LLM answer attempt (continues existing behavior)
   3. **Consult agents** — if LLM answer fails confidence threshold (or a predicate says so), invoke 1+ advisory tools via `ToolInvoker` (EA10-S7)
   4. **Synthesize consensus** — aggregate LLM + advisory outputs; produce a single candidate answer
   5. **Escalate** — if synthesis fails a confidence gate, terminal escalation (EventBus `supervisor.escalated`)

3. **AC3 — Confidence gating**
   - Each phase produces a confidence score (0–1)
   - Thresholds are configurable (`supervisor.confidence.llm`, `supervisor.confidence.synthesis` in `cop1.config.yaml`, sensible defaults)
   - Below threshold → advance to next phase; at threshold or above → accept answer, exit loop

4. **AC4 — `SupervisorContext` enrichment (EA11-S6)**
   - The resolution loop consumes `SupervisorContext` loaded by EA11-S6 — PRD, architecture, project metadata are available to LLM prompts and consultation queries

5. **AC5 — Auto-decision logging**
   - Every phase transition emits an `auto-decision` structured event (per EA10-S4 AC6)
   - Log captures: input question, phase traversed, intermediate outputs, final answer or escalation reason

6. **AC6 — Unit tests**
   - Scripted scenarios for:
     - Deterministic-resolvable (phase 1 returns with high confidence)
     - LLM-resolvable (phase 2 sufficient)
     - Consult-resolvable (phase 3 tips over threshold)
     - Escalation (all phases fail)
     - Context enrichment verified in LLM prompt payload
   - ≥ 6 test cases

7. **AC7 — Build gate**

## Dev Notes

### Risk #2 mitigation

Epics.md §EA10 Risk #2: **auto-decision loop robustness**. Mitigation = conservative LLM prompts (low temperature, strict JSON), `--step-by-step` required in early production runs, escalation easy. Encode these defaults.

### Confidence scoring — keep simple

- Deterministic match: confidence 1.0 if exact match, 0 otherwise
- LLM: use model's self-report (`0..1`) parsed from response — or bucket (low/med/high) mapped to (0.3/0.6/0.9)
- Synthesis: proportion of advisory agreements

Keep formulas transparent and logged.

### Escalation is a feature

Escalation is NOT a failure mode to avoid at all costs. It is the designed safety valve. Tune thresholds so escalation fires whenever the supervisor is genuinely uncertain.
