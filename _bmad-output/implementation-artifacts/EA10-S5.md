# Story EA10.5: `--step-by-step` mode (inter-command pauses)

Status: done

## Story

As a **cop1 operator running the orchestrator for the first time on a new playbook**,
I want a `--step-by-step` flag that pauses the orchestrator before each BMAD command transition for manual approval,
so that I can inspect state between commands, catch drift early, and gain confidence in the automation before trusting it unattended.

## Acceptance Criteria

1. **AC1 — Inter-command pause**
   - When orchestrator mode is `step-by-step`, `OrchestratorService` (EA10-S4) awaits approval from `StepByStepController` (EA11-S3) **between** commands (e.g., after `create-story` completes, before `dev-story` starts)
   - Distinct from EA8-S5 intra-command pause (inside a BMAD session) — this story is inter-command only

2. **AC2 — Approval UX**
   - The pause surfaces a prompt to stdout: `[step-by-step] Next: <command> for story <id>. Continue? (y/n/abort)`
   - Reads from stdin synchronously; non-TTY environments fallback to reading an approval file path from `COP1_APPROVAL_FILE` env var (for CI)
   - `y` → continue; `n` → skip this command; `abort` → clean halt (persist state, emit `orchestrator.aborted`)

3. **AC3 — Pending state persistence**
   - On pause, a `orchestrator-pending.yaml` snapshot is written to `.cop1/` with: `runId`, `currentStoryId`, `nextCommand`, `modeDuringPause`, `ts`
   - On resume (approval), the pending file is cleared

4. **AC4 — Mode propagation through `StepByStepController`**
   - `StepByStepController.setMode('inter')` or `'both'` activates the pause
   - `OrchestratorService.run({ mode: 'step-by-step' })` flows down through the controller

5. **AC5 — Unit tests**
   - `StepByStepController` tested for mode transitions (already covered in EA11-S3) — extend with inter-command approval flow
   - `OrchestratorService` tests: fixture playbook with `step-by-step=true`, scripted stdin, assert pause + resume

6. **AC6 — Build gate**

## Dev Notes

### Why not reuse EA8-S5 code path

EA8-S5 pauses **inside** a BMAD session (intra-command). This story pauses **between** BMAD sessions (inter-command). Different lifecycle, separate implementation — same controller handles both via mode flags.

### Pending file format

Minimal YAML — only the keys needed to resume. Do NOT duplicate full `sprint-status.yaml` content.

### CI path

Many users will want to run in CI with manual gates via a file-based approval mechanism. `COP1_APPROVAL_FILE` env var points to a file; orchestrator polls (with 1 s interval, 30 min timeout) until the file contains `y` / `n` / `abort`. Keep the polling logic minimal.
