## Dev Notes

### Why the automated test cannot assert real commits or `.cop1/history/` files

`commit_anchor` (EA12-S1) and the 3-track history (EA11-S8) are invoked **from within a BMAD session turn**, not from the orchestrator. The supervisor decides to call `commit_anchor` as a tool during `/bmad-bmm-dev-story`. Without the real SDK + claude CLI spinning up, no session turns → no tool calls → no commits, no history files.

An in-memory session adapter returns canned turn outcomes but does not execute the tool catalogue. So any integration test that stays in-memory must accept this gap. The gap is **closed only by the manual real-run** (`scripts/ea13-real-run.sh`).
