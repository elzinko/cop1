## Story

As a **developer reading `<STORY>.md`**,
I want the `## Status:` line in the story body to reflect the current `sprint-status.yaml` value after the orchestrator runs,
so that the story file and the YAML index never drift out of sync (closing EA12 retro action item #4).

## Acceptance Criteria

1. **AC1 — Status mirrored into story body**
   - `OrchestratorService.persistStatus` also rewrites `## Status: <anything>` in `<projectRoot>/_bmad-output/implementation-artifacts/<storyKey>.md` to match the new YAML status.
   - The operation is **idempotent**: running the orchestrator twice with the same final status does not change the file.

2. **AC2 — Non-existent story file is a no-op**
   - If `<storyKey>.md` does not exist, the mirror step is a silent no-op. No crash, no write, no error.

3. **AC3 — Only the status line is touched**
   - The regex matches only the first `## Status:` occurrence at the start of a line.
   - All other content (title, AC, tasks, dev notes) is preserved byte-for-byte.
   - Trailing whitespace handling: the new line has no trailing spaces.

4. **AC4 — Exported helper for direct testing**
   - A pure helper `mirrorStoryStatusInBody(body: string, nextStatus: string): string` is exported from `OrchestratorService.ts`, testable in isolation.

5. **AC5 — Unit test**
   - Unit tests cover: happy path (status changes), idempotent (same status), missing `## Status:` line (file unchanged or status line added — specify behaviour), malformed body (no blow-up).

6. **AC6 — Build + tests green**
   - `pnpm build` exit 0.
   - New test passes.
   - Full suite: pre-existing 9 failures still unchanged.

## Dev Notes

### Decision — missing `## Status:` line

Two options:
- A. Leave body unchanged (zero-side-effect policy). Simpler.
- B. Insert the line after the title (hard to predict position, risks inserting mid-table).

Choose **A**. Stories created by BMAD workflows always have a `## Status:` line at creation time; a missing line means either a hand-crafted story or an unrelated markdown file sharing the story's name. Do not guess where to insert.

### Why this is not a race with operator edits

The orchestrator writes to the body only on phase transitions — same time it writes the YAML. If an operator edits the body mid-run, the next transition clobbers only the status line (regex targets only `^## Status:`). All operator content survives.

### Completion Notes List

- `mirrorStoryStatusInBody` uses `^##\s+Status:\s*[^\n]*$` — matches a level-2 heading starting with `Status:` and stops at the end of line. Non-heading `Status: …` lines (no `##` prefix) are intentionally untouched per AC3.
- Regex is `/m` (multiline) without `/g` — catches only the first occurrence. Rationale: stories with multiple `## Status:` blocks are pathological; operator-inserted status references in prose are preserved.
- The `if (mirrored !== body)` guard in `persistStatus` avoids a spurious filesystem write (and its `mtime` bump) when the status didn't actually change. Matches the "zero-side-effect" policy from AC3.
- Policy A chosen for missing-status-line (see dev notes): leave unchanged. Tested in `mirrorStoryStatusInBody.test.ts` case 3. Covered in AC2 for missing files.
