# Story EA12.6: Track 2 extended format + session log aggregation (D4)

Status: done

## Story

As a **cop1 developer reviewing orchestrator runs**,
I want Track 2 exchange history to include horodated shell commands, return codes, stderr, decision method, blockers, and gate results ‚Äî anchored to real commit SHAs from `commit_anchor`,
so that post-run review gives a complete picture of what the orchestrator did and why.

## Acceptance Criteria

1. **AC1 ‚Äî Extended Track 2 fields**
   - Each Track 2 markdown file (`.cop1/history/{sprint}/{story}/`) includes:
     - Horodated shell commands (timestamp + command + working directory)
     - Return codes and stderr (truncated to 500 chars if longer)
     - Decision method (`deterministic` | `llm` | `consult` | `escalation`)
     - Blockers encountered (if any)
     - Gate results (pass/fail per gate, if applicable)

2. **AC2 ‚Äî Commit SHA anchoring**
   - When `commit_anchor` (EA12-S1) produces a commit, the SHA is recorded in the Track 2 entry for that step.
   - Track 2 files reference commit SHAs as `commit: <sha>` in their frontmatter.

3. **AC3 ‚Äî SessionLogAggregator integration**
   - The existing SessionLogAggregator (or transcript generator) consumes the extended Track 2 fields.
   - `cop1 transcript <session-id>` output includes the new fields in a readable format.

4. **AC4 ‚Äî Story‚Üîsession wiring**
   - Track 2 files reference the session ID in their frontmatter: `session: <session-id>`.
   - Debug logs reference sessions for cross-referencing.

5. **AC5 ‚Äî Tests**
   - Unit test verifying extended Track 2 format with all new fields.
   - Integration test verifying commit SHA appears in Track 2 after commit_anchor invocation.
