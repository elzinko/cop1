# Story EA10.3: cop1 minimal playbook (drives 1 epic)

Status: done

## Story

As a **cop1 operator running the Supervisor Orchestrator for the first time**,
I want a real, committed `supervisor-playbook.md` at the project root that drives a single epic loop (`create-story → dev-story → code-review` for each story of the target epic, no sprint-planning, no retrospective),
so that EA10-S4 has a concrete playbook to consume and the V1-light DoD "automate 1 epic end-to-end" can be demonstrated without authoring playbooks case by case.

## Acceptance Criteria

1. **AC1 — `supervisor-playbook.md` at project root**
   - File created at `/Users/elzinko/git/bacasable/cop1/supervisor-playbook.md`
   - Follows the spec produced in EA10-S2
   - Validates cleanly through `SupervisorPlaybookLoader` (EA10-S1)

2. **AC2 — Single loop: create-story → dev-story → code-review**
   - One phase per BMAD command, ordered list with the three commands
   - No sprint-planning, no retrospective (explicitly excluded — V1.1 scope)

3. **AC3 — Epic/story restrictions section**
   - Playbook is parameterized with `--epic <id>` at CLI invocation time (EA10-S6)
   - Restriction section explains: loop processes each story of the target epic in order of appearance in `sprint-status.yaml`, stopping at the first `done` / `cancelled` / explicit gate

4. **AC4 — Worktree hooks documented**
   - Section states that each story runs in its own worktree (reuses EA11-S3 `WorktreeService`)
   - Cleanup policy: worktree removed on success, kept on failure for debugging

5. **AC5 — Step-by-step hook documented**
   - Section states that `--step-by-step` (EA10-S5) pauses between the 3 commands per story

6. **AC6 — Decision policy**
   - Policy states: supervisor follows the 3-tier cascade (deterministic → LLM → escalate). Advanced auto-decision policies explicitly deferred to V1.1.
