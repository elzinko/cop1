# Story EA10.1: `SupervisorPlaybookLoader` — Markdown playbook parser

Status: done

## Story

As a **cop1 Supervisor Orchestrator about to start a 1-epic run**,
I want a markdown parser that reads `supervisor-playbook.md` and produces a structured list of phases and ordered commands with validated BMAD command references,
so that `OrchestratorService` (EA10-S4) can drive a canonical BMAD command sequence without re-parsing markdown or trusting untyped text each time.

## Acceptance Criteria

1. **AC1 — Loader exists with typed output**
   - Class `SupervisorPlaybookLoader` in `packages/app/src/features/orchestrator/application/SupervisorPlaybookLoader.ts`
   - `load(path: string): Promise<SupervisorPlaybook>`
   - `SupervisorPlaybook` type: `{ version: string, helpRef: string, phases: Phase[], epicRestrictions?: EpicRestrictions, hooks: { worktree?: string, stepByStep?: string }, decisionPolicy?: string }`
   - `Phase` type: `{ name: string, commands: { command: string, note?: string }[] }`

2. **AC2 — Parsing rules**
   - H2 sections (`## <phase name>`) map to `Phase` entries
   - Ordered lists (`1.`, `2.`, …) under each H2 map to commands, in order
   - Preamble (content before the first H2) is parsed for:
     - BMAD version line (regex: `BMAD version: (.+)`)
     - help command pointer (regex: `help: (/.+)`)
     - epic/story restrictions (optional section)
     - worktree / step-by-step hooks (optional)
     - decision policy (optional)

3. **AC3 — BMAD command reference validation**
   - Each extracted command (e.g., `/bmad-bmm-create-story`) is validated against the set of discovered slash commands in `.claude/commands/`
   - Unknown commands produce a `PlaybookValidationError` listing the offending command + its line number
   - Empty or missing phases produce the same error

4. **AC4 — Unit tests with sample playbooks**
   - Test file `packages/app/src/features/orchestrator/__tests__/SupervisorPlaybookLoader.test.ts`
   - Fixtures: a minimal valid playbook (1 phase, 2 commands), a playbook with all optional sections, a malformed playbook (missing H2), a playbook with an unknown BMAD command
   - ≥ 6 test cases

5. **AC5 — Exported via package barrel**
   - Exported from `@cop1/app` where the orchestrator feature lives

6. **AC6 — Build gate**
   - `pnpm typecheck` / `pnpm test` / `pnpm lint` pass, zero regressions
