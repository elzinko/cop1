# Story EA12.7: Aspirational features audit — delete/wire burndown, velocity, kpis (A7)

Status: todo

## Story

As a **cop1 maintainer**,
I want dead or unwired aspirational feature modules audited and either deleted or wired to real consumers,
so that the codebase reflects actual capabilities and does not carry misleading dead code.

## Acceptance Criteria

1. **AC1 — kpis-dashboard deleted**
   - `kpis-dashboard/` feature module deleted (zero functional consumer — barrel re-export only per SCP §A7).
   - All barrel re-exports referencing it cleaned up.

2. **AC2 — burndown/ decision**
   - `burndown/` module either:
     - (a) Wired to a cop1 command that consumes `/bmad-bmm-sprint-status` output and produces burndown data, OR
     - (b) Deleted with a backlog item for V1.2 if no concrete consumer exists now.
   - Decision recorded in this story's completion notes.

3. **AC3 — velocity-projector/ decision**
   - `velocity-projector/` module either:
     - (a) Wired to a cop1 command, OR
     - (b) Deleted with a backlog item for V1.2.
   - Decision recorded in this story's completion notes.

4. **AC4 — No orphan imports**
   - After deletions, `pnpm build` succeeds with zero import errors.
   - `pnpm test` passes.

## Tasks / Subtasks

- [ ] Task 1 — Audit consumers (AC: #1, #2, #3)
  - [ ] Grep for imports of kpis-dashboard, burndown, velocity-projector
  - [ ] Determine which have real consumers vs barrel-only

- [ ] Task 2 — Delete kpis-dashboard (AC: #1)
  - [ ] Remove module directory

- [ ] Task 3 — burndown + velocity-projector decision (AC: #2, #3)
  - [ ] If hookable to /bmad-bmm-sprint-status, wire minimally

- [ ] Task 4 — Build + test verification (AC: #4)
