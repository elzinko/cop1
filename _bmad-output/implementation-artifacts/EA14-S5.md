# Story EA14-S5: Investigate + fix 9 pre-existing SprintRunner test failures

Status: done

## Summary

Fixed 9 pre-existing test failures across 2 test files:
- `packages/app/src/composition/__tests__/SprintRunner.test.ts` (8 tests)
- `packages/app/src/integration-tests/bmad-pipeline-e2e.test.ts` (1 test)

## Root Cause

After EA9/EA11, `BMADReader.listStories()` was updated to scan
`_bmad-output/implementation-artifacts/` for story `.md` files (flat layout,
matching `E[A]?\d+-S\d+` filename pattern). However, both test files still
created their fixture stories under the old path
`_bmad-output/planning-artifacts/stories/sprint-0/`. This caused
`BMADReader.listStories()` to return an empty array, so `eligibleStories`
was always empty and `storiesDone` was always 0.

## Fix

Updated the `createTempProject()` helpers in both test files to place story
fixtures in `_bmad-output/implementation-artifacts/` where `BMADReader`
actually looks.

### Files Changed

- `packages/app/src/composition/__tests__/SprintRunner.test.ts` -- changed story fixture path
- `packages/app/src/integration-tests/bmad-pipeline-e2e.test.ts` -- changed story fixture path

## Verification

All 13 tests now pass (11 in SprintRunner.test.ts + 2 in bmad-pipeline-e2e.test.ts).
