# Story EA11.7: Session transcript generator + `cop1 transcript` CLI

Status: done

## Story

As a **Developer who needs to audit an orchestrator run**,
I want a session-level human-readable markdown transcript automatically composed from Track 2 exchange files, accessible via a `cop1 transcript <session-id>` CLI command,
so that I can review the full Q/A + tool calls + decisions of a multi-turn BMAD session without parsing JSONL or SSE streams, and so that transcripts are diffable, committable, and shareable.

## Acceptance Criteria

1. **AC1 — `SessionTranscriptGenerator` component**
   - New class `SessionTranscriptGenerator` in `packages/observability/src/features/transcript/application/SessionTranscriptGenerator.ts`
   - Method `generate(sessionId: string): Promise<string>` returns a single markdown document
   - Reads Track 2 files via `ExchangeHistoryReader` (from EA11-S8), sorts by timestamp, composes a session-level document

2. **AC2 — Transcript structure**
   - Top header: session ID, sprint/story, start/end, duration, command list, outcome
   - One section per BMAD command invocation (H2) containing:
     - Command name + timestamp
     - Supervisor Q/A in chronological order (distinct blockquotes for Q and A)
     - Tool invocations with name + truncated input/output (max 2 KB per call, with full-output pointer to the source file)
     - System events (start/end/errors/retries) in a collapsible summary block
   - Footer: links to source Track 2 files
   - Markdown passes `markdownlint` with default rules (or existing cop1 config)

3. **AC3 — CLI `cop1 transcript <session-id>`**
   - New CLI subcommand in `packages/app/src/cli/` wires `SessionTranscriptGenerator`
   - Flags: `--out <path>` (default: stdout), `--regenerate` (re-read from disk rather than cache), `--open` (open in `$EDITOR` after generation, UX convenience)
   - Exit codes: 0 success, 1 session not found, 2 read error
   - Discoverable via `cop1 --help`

4. **AC4 — Regeneration is idempotent**
   - Running `cop1 transcript <id>` twice on unchanged Track 2 files produces byte-identical output (deterministic ordering + no timestamps in the output body beyond source data)

5. **AC5 — Missing session handling**
   - When no Track 2 files match `sessionId`: clear error to stderr `No transcript data for session <id>`, exit 1

6. **AC6 — Unit + integration tests**
   - `SessionTranscriptGenerator.test.ts` — composes transcript from fixture Track 2 files, covers structure, tool truncation, sort order
   - CLI integration test — invokes `cop1 transcript <id>` against a fixture `.cop1/history/` tree, asserts stdout content

7. **AC7 — Build gate**
   - `pnpm typecheck` / `pnpm test` / `pnpm lint` pass zero regressions.

## Dev Notes

### Hard dependency on EA11-S8

This story cannot start until EA11-S8 is merged because `ExchangeHistoryReader` is the data source.

### Package ownership

Place in `@cop1/observability` (existing package per architecture). Keep the generator free of any CLI-specific code — CLI imports the library. `ExchangeHistoryReader` is imported from `@cop1/sprint-core` — produced by EA11-S8 in `packages/sprint-core/src/features/bmad-orchestration/infrastructure/`.

### Deterministic output

No wall-clock timestamps in rendered output beyond what's in the source front-matter. Sort by `started_at` then by filename as tiebreaker for stable diffs.

### Tool I/O truncation

Tool call inputs/outputs can be large (file contents, command output). Truncate at 2 KB and append `… [truncated, full output in <path>]` pointing to the Track 2 source file. Keeps transcripts readable.

## Senior Developer Review (AI)

**Reviewer:** Claude Opus 4.6 | **Date:** 2026-04-14 | **Outcome:** Approved with documented deviations

Deviations from original story spec:
1. **Package location:** `sprint-core` instead of `@cop1/observability`. Rationale: `observability` depends on nothing, `sprint-core` already depends on observability — putting the transcript generator in observability would create a circular dep. Documented in source JSDoc.
2. **CLI flags:** `--regenerate` and `--open` dropped. `--regenerate` is the default (no caching yet); `--open` is a UX convenience with no behavioral value in CI. Trivial to restore.

Core AC coverage intact: transcript structure (header/sections/footer), deterministic output, session-not-found handling, CLI entry point. No critical findings.
