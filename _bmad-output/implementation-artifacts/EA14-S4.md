# Story EA14-S4: SupervisorContext name clash resolution

Status: done

## Story

As a **developer importing types from `@cop1/sprint-core`**,
I want the per-question context type to have a distinct, unambiguous name (`SupervisorQuestionContext`),
so that it cannot be confused with the EA11-S6 bootstrap `SupervisorContext` (prd, architecture, projectMetadata).

## Acceptance Criteria

1. **AC1 — Rename in source**
   - The `SupervisorContext` interface in `SupervisorLLMPort.ts` is renamed to `SupervisorQuestionContext`.

2. **AC2 — Barrel updated**
   - `SupervisorQuestionContext` is exported directly from `@cop1/sprint-core` barrel.
   - The `SupervisorAnswerContext` alias is removed.
   - The bootstrap `SupervisorContext` (from `domain/SupervisorContext.ts`) is unaffected.

3. **AC3 — All consumers updated**
   - All imports and usages in sprint-core and app packages reference the new name.

4. **AC4 — `pnpm build` green**

5. **AC5 — Existing tests pass (no regression)**

## Dev Notes

### Implementation choice

The per-question context (used by `SupervisorService.setWorkflowContext` and `SupervisorLLMPort.generateResponse`) was renamed from `SupervisorContext` to `SupervisorQuestionContext`. This eliminates the naming collision with the EA11-S6 bootstrap `SupervisorContext` (which holds prd, architecture, projectMetadata, iamthelaw, loadedAt).

The intermediate alias `SupervisorAnswerContext` (added in EA13-S2 as a barrel workaround) is removed -- the type is now exported directly under its canonical name.
