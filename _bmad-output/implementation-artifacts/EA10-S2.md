# Story EA10.2: Playbook format specification + reference example

Status: done

## Story

As a **cop1 contributor writing or reviewing a supervisor playbook**,
I want a formal specification of the required sections and a committed reference example of a valid playbook,
so that authoring a new playbook is unambiguous and regressions in playbook parsing surface against a canonical fixture.

## Acceptance Criteria

1. **AC1 — Spec document**
   - New doc `_bmad-output/planning-artifacts/supervisor-playbook-format.md`
   - Sections: overview, required sections, optional sections, grammar rules (H2/ordered list mapping), validation rules, example

2. **AC2 — Required sections documented**
   - BMAD version preamble (e.g., `BMAD version: 6.0.0-Beta.8`)
   - `/bmad-help` pointer
   - Process sequence (≥ 1 H2 phase with ≥ 1 command)
   - Epic/story restrictions (optional but format specified)
   - Worktree hooks (optional, format specified)
   - Step-by-step hooks (optional, format specified)
   - Decision policy (optional, format specified)

3. **AC3 — Reference example committed**
   - New file `_bmad-output/planning-artifacts/supervisor-playbook-reference.md`
   - Complete valid playbook exercising every section
   - Used as a positive test fixture in EA10-S1

4. **AC4 — Grammar rules explicit**
   - Each rule references the parsing behavior in `SupervisorPlaybookLoader` (EA10-S1)
   - Ambiguities resolved (e.g., what happens if H2 has no ordered list? → validation error)

5. **AC5 — Cross-linked**
   - `supervisor-playbook-format.md` linked from epics.md §EA10 and from the architecture doc
   - `supervisor-playbook-reference.md` linked from the format doc
