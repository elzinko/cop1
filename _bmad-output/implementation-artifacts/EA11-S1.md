## Story

As a **cop1 maintainer**,
I want the legacy standalone cop1 agent classes (DevAgent, ReviewerAgent, QAAgent, PMAgent + their `*Step` wrappers) marked `@deprecated` with a rationale pointing to the BMAD-driven replacement path,

## Dev Notes

### Why deprecation, not deletion

Per SCP 2026-04-11 and epics.md §EA11, the legacy agent classes are the **safety net** while EA10 Supervisor Orchestrator is brought up. Deletion is explicitly deferred until after EA10-S9 integration test passes on the EA6 cobaye fixture. Deprecating now signals intent to new contributors without removing the fallback.

The `config.workflow.useBMAD=false` runtime path (deprecated in EA11-S2) still routes through `DevAgentStep`/`ReviewerAgentStep`/`QAAgentStep`/`PMAgentStep`, so removing them breaks the fallback. Deprecation is the right level.

### `@deprecated` JSDoc format

Standard TSDoc `@deprecated` tag format — Biome accepts it, and TypeScript emits `deprecated` diagnostics for consumers automatically (strikethrough in IDEs).

Template to use consistently on each class:

```typescript
/**
 * @deprecated Since 2026-04-14 (EA11-S1). The standalone cop1 agent pattern
 * is superseded by BMAD-driven multi-turn sessions invoked through
 * {@link BMADSessionPort} (ADR-012). This class remains as a safety-net
 * fallback while `config.workflow.useBMAD=false` is still supported
 * (deprecated in EA11-S2). Migration path: BMAD `<command>` workflow
 * driven by `SupervisorService` / `OrchestratorService` (EA10).
 * Scheduled for removal once EA10-S9 integration test passes in production.
 */
export class DevAgent implements WorkflowStep {
  // ...
}
```

Replace `<command>` per agent: `dev-story` (DevAgent, DevAgentStep), `code-review` (ReviewerAgent, ReviewerAgentStep), `qa` (QAAgent, QAAgentStep), `create-story` / PM orchestration (PMAgent, PMAgentWorkflowStep, PMAgentStep).

3. **[LOW] F3 — JSDoc template drops `{@link BMADSessionPort}` syntax from the story's template.** Original template used TSDoc `{@link}` markers; my final JSDoc uses inline backticks instead. Both are valid; `{@link}` gives IDE jump-to-definition but requires the type to be imported in scope — these files do not import `BMADSessionPort`, so backticks are functionally equivalent without triggering import churn. Deferred.

4. **[LOW] F4 — No explicit guard that barrel exports (`index.ts`) keep re-exporting the deprecated classes.** AC4 is enforced by the fact that no barrel file was touched + existing import sites (e.g., `PipelineStepFactory`) still typecheck.
