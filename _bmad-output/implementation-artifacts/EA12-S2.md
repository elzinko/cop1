# Story EA12.2: Recovery E2E test with real AgentSdkSessionAdapter

Status: done

## Story

As a **cop1 release gatekeeper**,
I want an E2E test that exercises Strategy A (SDK `resume: session_id`) with the real `AgentSdkSessionAdapter` (not `InMemorySessionAdapter`),
so that session recovery is validated before V1.1 "dogfooding unassisted" sign-off.

## Acceptance Criteria

1. **AC1 — Real adapter used**
   - Test instantiates `AgentSdkSessionAdapter` (not `InMemorySessionAdapter`).
   - Session is created, a BMAD command is run, session is suspended, then resumed via `resume: session_id`.

2. **AC2 — Recovery validated**
   - After resume, the session state (context, history, pending operations) is consistent with pre-suspension state.
   - A follow-up BMAD command executes successfully on the resumed session.

3. **AC3 — Crash recovery scenario**
   - Test simulates an abrupt session termination (process kill or adapter disconnect).
   - Recovery path re-instantiates the adapter and resumes from the last checkpoint.

4. **AC4 — CI-runnable**
   - Test runs under `pnpm test:e2e` or equivalent.
   - Uses mocked LLM responses (no real API calls) but real SDK session lifecycle.

5. **AC5 — Migration from Plan B**
   - EA10-S9's `InMemorySessionAdapter` fixture is either replaced or supplemented by this real-adapter test.
   - The sentinel marker left in EA10-S9 for this migration is resolved.

### Completion Notes List

- Two new public methods on `AgentSdkSessionAdapter` unlock the crash-recovery path:
  - `getSdkSessionId(localSessionId)`: accessor so callers can persist the SDK-assigned session_id between crashes.
  - `restoreSession(sdkSessionId, context)`: fresh-adapter entrypoint that primes the internal maps with the persisted id and returns a new local sessionId. Subsequent `continueSession` calls flow `resume: <sdkSessionId>` to the SDK query.
- The test uses a capturing `QueryFunction` that records per-invocation options — verifying `resume: ...` flows end-to-end in both the single-adapter and cross-adapter scenarios. No real API calls.
- **Note on `InMemorySessionAdapter`**: not deleted — it's still useful as a lightweight fixture for tests that don't exercise the SDK path. EA10-S9's recovery-specific reliance on it is what's retired; general fixture usage continues.
