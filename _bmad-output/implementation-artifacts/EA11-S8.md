# Story EA11.8: Refactor `SessionLogger` / `SessionHistoryReader` → 3-tracks file-based structure

Status: done

## Story

As a **cop1 Developer who wants git-visible session artefacts**,
I want `SessionLogger` and `SessionHistoryReader` refactored so that every BMAD command invocation writes to a file-based 3-tracks structure (Track 1 SDK session, Track 2 exchange markdown committed to git, Track 3 metrics JSONL gitignored),
so that EA11-S7's transcript generator can read committable markdown files directly, metrics stay separable from content for dashboards, and the dead `sprint-journal/` feature gets cleaned up in the same pass.

## Acceptance Criteria

1. **AC1 — Track 2 writer produces exchange markdown**
   - New class `ExchangeHistoryWriter` in `packages/sprint-core/src/features/bmad-orchestration/infrastructure/ExchangeHistoryWriter.ts`
   - Writes one markdown file per BMAD command invocation at `.cop1/history/{sprintId}/{storyId}/{timestamp}-{command}.md`
   - File front-matter (YAML): `session_id`, `story_id`, `sprint_id`, `command`, `started_at`, `ended_at`, `supervisor_turns`, `status`
   - Body: chronological Q/A (supervisor questions + answers), system events (start/end/errors/retries), tool invocations (name + truncated input/output)
   - Front-matter injection is escaped so markdown content containing `---` does not break parsing
   - Files are committed to git by default

2. **AC2 — Track 3 writer produces metrics JSONL**
   - New class `MetricsWriter` in `packages/sprint-core/src/features/bmad-orchestration/infrastructure/MetricsWriter.ts`
   - Appends one JSON line per metric event to `.cop1/metrics/YYYY-MM-DD.jsonl`
   - Metric schema: `{ ts, sessionId, storyId, sprintId, event, ...data }` — covers `session.started`, `session.turn.completed`, `session.workflow.completed`, `session.workflow.failed`, plus token/latency data from `llm.call.completed`
   - Daily file rotation by local date

3. **AC3 — `ExchangeHistoryReader` for Track 2**
   - New class `ExchangeHistoryReader` in `packages/sprint-core/src/features/bmad-orchestration/infrastructure/ExchangeHistoryReader.ts`
   - Reads Track 2 files for a given `sessionId` or `storyId`
   - Parses front-matter + body
   - Exposes `byStory(storyId)`, `bySession(sessionId)`, `listAll()`

4. **AC4 — `SessionLogger` refactored as router**
   - Existing `SessionLogger` now routes each recorded interaction to BOTH `ExchangeHistoryWriter` (Track 2) AND `MetricsWriter` (Track 3) where appropriate
   - Public `SessionInteraction` type preserved (no breaking change to callers)
   - Existing `EventBus` integration preserved (SSE consumers continue to receive events)

5. **AC5 — `SessionHistoryReader` refactored**
   - Existing `SessionHistoryReader` now delegates Track 2 reads to `ExchangeHistoryReader`
   - Any consumer of `SessionHistoryReader` keeps working (public API preserved)

6. **AC6 — `.gitignore` bootstrap guard**
   - On first run, cop1 detects `.cop1/metrics/` is not in `.gitignore` and adds it (+ any other gitignored tracks)
   - Gate: only mutates `.gitignore` when a `--yes` flag is passed OR when an idempotent detection confirms no prior config
   - Never duplicates entries
   - Covered by ADR-014 §8.6

7. **AC7 — `sprint-journal/` feature deleted**
   - Directory + all associated code removed (feature declared dead per SCP 2026-04-11 §4 addendum)
   - No references remain in composition root, tests, or config schema
   - Related tests are removed, not migrated

8. **AC8 — Unit + integration tests**
   - `ExchangeHistoryWriter.test.ts`, `MetricsWriter.test.ts`, `ExchangeHistoryReader.test.ts` — each ≥ 4 cases
   - Update `SessionLogger.test.ts` to assert dual routing (Track 2 + Track 3)
   - Integration test: full BMAD command simulated → both Track 2 markdown and Track 3 JSONL produced with expected content

9. **AC9 — Build gate**
   - `pnpm typecheck` / `pnpm test` / `pnpm lint` all pass, zero regressions on remaining tests.

## Dev Notes

### Capacity watch

Epics.md flags this story as **Medium-Large, ~950–1350 LOC with tests**. Sprint 12 capacity is a concern — EA11-S1/S2 (XS deprecations) can slip to Sprint 13 if needed since they unblock nothing critical.

### Architectural reference

ADR-014 §8.5 and §8.6 define the exact structure. Do NOT re-decide structural questions in this story; implement what is already decided.

### Front-matter escaping

If a Q/A body contains a line that starts with `---`, escape it as `\---` or wrap the body section after a sentinel line. Pick one and document it in the writer's JSDoc.

### Atomic writes

Use `fs.writeFile(tmp)` + `fs.rename(tmp, target)` — avoids partial files if the process crashes mid-write. Critical for committable artefacts.

### SessionInteraction preserved

Callers (SSE consumers, tests) import `SessionInteraction` from the bmad-orchestration barrel. Keep the re-export path stable.

### Delete, don't migrate, `sprint-journal/`

Per SCP decision. Migration has zero value since the directory is declared dead before this refactor ships.

- `MetricsWriter` called fire-and-forget (`void`) from `SessionLogger` — metrics are best-effort, not transactional with the primary logger
- Front-matter escape uses HTML comment sentinel (`<!-- cop1:body-start -->`) — unambiguous parser signal, invisible in rendered markdown
- `GitignoreBootstrap` fix applied: idempotency detection keys only on path entries (not the managing comment), so a user who pre-populated `.cop1/metrics/` doesn't get a redundant comment/entry pair appended
