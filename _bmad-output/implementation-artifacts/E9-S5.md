# Story 9.5: Rule Approval UI

Status: ready-for-dev

## Story

As a Developer,
I want a Web UI to consult, approve, modify, or reject rule proposals from agents,
so that I can review improvement suggestions and apply them to the project governance.

## Acceptance Criteria

1. Web UI displays a list of all rule proposals from `RuleProposalService.getAll()`, organized by type (architecture, team, agent, quality, process) with status badges (pending/approved/rejected/debated)
2. Each proposal card shows: ruleId, type, description, reason, submittedBy, submittedAt, status
3. Developer can approve (triggers `RuleProposalService.updateStatus(id, 'approved')` + emits `rule.applied` event), reject (with reason text field), or send to debate (triggers `rule.proposal.debated` event)
4. API endpoint `GET /api/rules/proposals` returns all proposals; `PATCH /api/rules/proposals/:id` updates status
5. Real-time updates via SSE — new proposals appear without page refresh

- [ ] Task 3: SSE integration (AC: #5)
  - [ ] Subscribe to `rule.proposal.submitted` and `rule.applied` events in React

## Dev Notes

### Architecture Patterns

- **HttpServer** (`packages/app/src/features/daemon/infrastructure/HttpServer.ts`): Add routes following existing pattern (GET /health, GET /api/sprint/status). Use `req.params` for `:id`.
- **RuleProposalService** (`packages/sprint-core/src/features/rule-proposal/application/RuleProposalService.ts`): Already has `getAll()`, `getById()`, `updateStatus()` methods. In-memory Map storage.
- **EventBus** (`packages/shared-kernel/src/features/events/domain/EventBus.ts`): Emit via `eventBus.emit(eventType, payload)`. SSE bridge in HttpServer automatically forwards to `/events`.
- **Web app** (`packages/web/src/App.tsx`): Currently has 3 tabs (Projects, Agents, Tasks). Add a 4th tab "Rules" or integrate into existing structure.

### Project Structure Notes

- API routes go in `HttpServer.ts` — no separate router files in current architecture
- React components in `packages/web/src/` — flat structure, no component subdirectories currently
- Follow existing dark theme design system: slate-900 bg, slate-800 cards, blue-400 accent
- Status colors: pending=amber-400, approved=emerald-400, rejected=red-400, debated=blue-400
